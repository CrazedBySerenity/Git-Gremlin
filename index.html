The Git Gremlin

A guide to git by the git gremlin himself!

  • Who is he?
  • What does he do?
  • What can he do for you?

Welcome to the git gremlins guide to git. The git gremlin is a very nice guy who sits in your project when you use git to help record and manage your changes. Here he has provided you with a guide to help you make the best use of his services!
If you'd like to know more about the gremlin species that the git gremlin comes from, here is a link to a wikipedia article with more information

When you initialise git in your project, you will summon the git gremlin who will start writing down all your changes in his trusty notebook. Then, as you go along he will make note of your changes, help you organise them and when it's time he will help you send them to others!
But at this point you're probably thinking to yourself this all sounds great, but how can I make use of this? Fear not! Because the git gremlin has made this handy guide for you!
  • git init
    •  Creates the .git folder
    Git gremlin is summoned and starts writing down all of the changes you make
  • git add
    •  Specifies a file that will be included in the next commit
    Git gremlin is told that a certain file is a cool file and he should share it with his friends in the future
  • git commit
    •  Saves a copy of your project (basically) with the changes made so far for later use
    Git gremlin saves the current notebook he's been using in a safe box for later use. PS. Only cool files allowed!
  • git push
    •  Uploads your changes (in the commit) to github
    Git gremlin tells all of his friends about the new changes that have been made. PS. Only cool files allowed!
Of course there's more that the git gremlin can do, but for now that should get you started!
